# FlowLocal

Локальная диктовка для macOS в духе Wispr Flow — полностью офлайн, бесплатно,
на [whisper.cpp](https://github.com/ggerganov/whisper.cpp) с Metal-ускорением
для Apple Silicon.

**Как это работает:** приложение висит в menu bar. В любом окне зажимаете
**правый Option (⌥)**, диктуете (по-русски или по-английски), отпускаете —
текст появляется в поле ввода, где стоял курсор. Telegram, Chrome, VS Code,
Notes — любое приложение. Аудио и текст никуда не отправляются.

## Требования

- macOS 14+ (Sonoma) на Apple Silicon (M1 и новее);
- Xcode Command Line Tools (`xcode-select --install`) или Xcode;
- `cmake` и `git` (`brew install cmake`);
- ~2 ГБ на диске для модели large-v3-turbo (или ~500 МБ для small).

Интернет нужен один раз: скачать whisper.cpp при сборке и модель при первом
запуске. Дальше всё работает офлайн.

## Сборка и запуск

```bash
cd FlowLocal

# 1. Собрать whisper.cpp с Metal (один раз, ~2–3 минуты)
./scripts/build-whisper.sh

# 2. Собрать и упаковать приложение
./scripts/make-app.sh

# 3. Установить
mv dist/FlowLocal.app /Applications/
open /Applications/FlowLocal.app
```

Либо откройте `Package.swift` в Xcode и запустите таргет `FlowLocal`
(шаг 1 всё равно обязателен — он создаёт `Frameworks/whisper.xcframework`).

Для быстрой отладки подойдёт `swift run`, но автозапуск при входе в систему
работает только у собранного `.app`.

## Разрешения macOS (обязательно)

При первом запуске приложение само запросит три разрешения. Если что-то не
выдалось — меню-бар → «Настройки…» → раздел «Разрешения», там видно, чего не
хватает, с кнопками перехода в нужную панель:

| Разрешение | Где выдать | Зачем |
|---|---|---|
| Микрофон | Настройки → Конфиденциальность → Микрофон | запись голоса |
| Универсальный доступ | … → Универсальный доступ | эмуляция Cmd+V и перехват клавиши |
| Мониторинг ввода | … → Мониторинг ввода | глобальный перехват правого ⌥ |

После выдачи «Универсального доступа» и «Мониторинга ввода» перезапустите
приложение (или нажмите «Обновить статус разрешений» в настройках).

**Важно:** после пересборки приложения macOS может сбросить разрешения —
это нормально для ad-hoc подписи, просто выдайте их заново.

## Модели

При первом запуске автоматически скачивается выбранная модель с HuggingFace
(официальные веса `ggerganov/whisper.cpp`) в
`~/Library/Application Support/FlowLocal/models/`.

| Модель | Размер | Комментарий |
|---|---|---|
| `large-v3-turbo` | ~1.6 ГБ | по умолчанию: быстрая и лучшая для русского |
| `medium` | ~1.5 ГБ | медленнее turbo, качество сопоставимо |
| `small` | ~488 МБ | самая лёгкая, слабее на русском |

Модель грузится в память один раз при старте и прогревается холостым прогоном,
поэтому короткая фраза на прогретой модели распознаётся примерно за секунду.

## Настройки (menu bar → Настройки…)

- **Модель** — small / medium / large-v3-turbo, смена перезагружает модель;
- **Язык** — авто (по умолчанию) / фиксированный русский / английский;
- **Режим клавиши** — hold-to-talk (по умолчанию) или toggle
  (нажал — старт, нажал ещё раз — стоп);
- **Вставка** — через буфер обмена с восстановлением (по умолчанию) или
  посимвольный ввод (fallback для приложений, где Cmd+V перехвачен);
- **AI-очистка** (выкл по умолчанию) — прогон текста через локальный Ollama:
  убирает слова-паразиты и расставляет пунктуацию. Нужен запущенный
  `ollama serve` и модель (`ollama pull qwen2.5:3b`). Всё локально; если
  Ollama недоступен, текст вставляется как есть;
- **Автозапуск** — запуск при входе в систему (SMAppService, только для .app).

Горячая клавиша фиксирована: **правый Option** (keyCode 61). Левый Option
работает как обычно.

## Архитектура

```
FlowLocalApp (SwiftUI MenuBarExtra + Settings)
└── DictationController  — оркестратор пайплайна и состояния
    ├── HotkeyManager    — CGEventTap: правый ⌥ (61) vs левый (58); hold/toggle;
    │                      глотает события клавиши, чтобы не печатались спецсимволы
    ├── AudioRecorder    — AVAudioEngine + AVAudioConverter → 16 kHz mono Float32
    ├── Transcriber      — actor поверх whisper.cpp (C API); модель в памяти,
    │                      прогрев при старте; Metal (use_gpu, flash_attn)
    ├── TextCleanup      — опц. очистка через локальный Ollama (127.0.0.1:11434)
    ├── TextInjector     — снапшот pasteboard → текст → Cmd+V → восстановление;
    │                      fallback: keyboardSetUnicodeString
    ├── OverlayController — non-activating NSPanel поверх всех окон:
    │                      «Слушаю…» / «Распознаю…» + уровень микрофона
    ├── PermissionsManager — микрофон / Accessibility / Input Monitoring
    ├── ModelManager     — скачивание и хранение ggml-моделей
    └── AppSettings      — UserDefaults-настройки
```

Поток данных: `правый ⌥ вниз` → AudioRecorder.start → `⌥ вверх` →
PCM 16 kHz → Transcriber.transcribe → (TextCleanup) → TextInjector.inject.

Где что менять:

- другая горячая клавиша → `HotkeyManager.swift` (константы keyCode);
- параметры распознавания (beam search, потоки, prompt) → `Transcriber.swift`;
- список моделей / источник скачивания → `ModelManager.swift`;
- версия whisper.cpp → `scripts/build-whisper.sh` (`WHISPER_VERSION`);
- вид индикатора → `OverlayController.swift` (`OverlayView`).

Почему прямой C-bridge вместо SwiftWhisper: SwiftWhisper давно не обновлялся,
привязан к старому whisper.cpp без Metal и не знает large-v3-turbo. Сборка
свежего whisper.cpp в xcframework (см. `scripts/build-whisper.sh`) даёт Metal,
актуальные модели и полный контроль над параметрами.

## Приватность

- аудио живёт только в оперативной памяти, на диск не пишется;
- распознанный текст не логируется и не сохраняется;
- сеть используется только для разового скачивания модели и (если включено)
  запросов к локальному Ollama на 127.0.0.1;
- буфер обмена полностью восстанавливается после вставки (все типы данных,
  включая картинки и rich text).

## Известные ограничения

- Правый Option занят приложением целиком: комбинации «правый ⌥ + клавиша»
  (спецсимволы на некоторых раскладках) перестают работать, пока FlowLocal
  запущен. Левый Option не затронут.
- Менеджеры буфера обмена (Paste, Maccy и т.п.) увидят вставляемый текст —
  это неизбежно для clipboard-режима; используйте посимвольный режим, если
  это критично.
- Ad-hoc подпись: после пересборки macOS просит выдать разрешения заново;
  для распространения нужна подпись Developer ID + notarization.
- В полях с защищённым вводом (пароли) macOS блокирует event tap — это
  системное поведение, диктовка туда не работает.
- Очень длинная диктовка (минуты) распознаётся одним куском после отпускания
  клавиши — стриминга «по мере речи» пока нет.
- Автоопределение языка whisper иногда ошибается на коротких фразах со
  смешанной речью — зафиксируйте «Русский» в настройках, если диктуете
  в основном по-русски.

## Идеи для доработок

- стриминговое распознавание во время записи (whisper_state + чанки);
- настраиваемая горячая клавиша (правый Cmd, Fn, двойное нажатие);
- пользовательский словарь / initial prompt для терминов и имён;
- VAD-отсечка тишины для ускорения на длинных записях;
- история диктовок (строго опционально, с явным согласием);
- поддержка Intel-маков (сборка без Metal, CPU-режим);
- Developer ID подпись и notarization для раздачи готового .app.
